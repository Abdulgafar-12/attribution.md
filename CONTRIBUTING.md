# Contributing to ATTRIBUTION.md

Thanks for your interest in improving the protocol. This is an open convention and contributions are welcome from maintainers, AI agent developers, platform operators, and the broader open source community.

## How to Contribute

**Spec feedback or questions:** [Open an issue](https://github.com/attributionmd/attribution.md/issues). Describe the problem or suggestion clearly. Reference specific sections of the specification where relevant.

**Wording or structural improvements:** Submit a pull request against `main`. Keep changes focused. Explain what changed and why in the PR description.

**Discussion:** Use [GitHub Issues](https://github.com/attributionmd/attribution.md/issues) for open-ended discussion about the protocol's direction, new action types, or adoption strategy.

## What We're Looking For

- Clarifications to ambiguous specification language
- Edge cases not covered by the current parsing rules
- Feedback from AI agent developers on implementation feasibility
- Feedback from maintainers on adoption barriers
- Corrections to the FAQ or documentation

## Guidelines

- Keep proposals grounded in the [Non-Goals](README.md#4-non-goals) section. The protocol is intentionally narrow.
- Changes to normative language (MUST, SHOULD, MAY) require clear justification.
- See [GOVERNANCE.md](GOVERNANCE.md) for versioning policy and decision-making guidelines.
